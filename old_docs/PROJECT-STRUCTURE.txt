WHAT EACH COMPONENT DOES:

VIEWS (Main Pages):
  index.html    → Landing with auth, FSAT info, success stories
  test.html     → FSAT test with onboarding, auth protection
  resume.html   → Quick 5-min resume with Claude AI
  testscript.js → Test logic, auto-save, anti-cheat, API calls

PUBLIC (Assets):
  logo.png           → FutureTech logo (Red & Black)
  supabase-auth.js   → Google OAuth, session management

CONFIG:
  questions.json → 18 test questions with marks

FRONTEND (React App):
  BuilderPage      → Advanced resume builder UI
  ResumeSelectorPage → User's saved resumes
  ProfilePage      → User profile management
  LoginPage        → Login form
  Uses Supabase for auth, proxied through main server

RESUME (Python Service):
  server.py        → FastAPI endpoints (/process, /generate)
  Agents           → AI agents for parsing, enhancing
  Generators       → PDF/HTML generation
  Uses Groq LLM for advanced AI features

SERVER.JS (Main):
  - Serves views/ pages
  - Serves React app at /builder/*
  - Proxies to Python service at /api/resume/*
  - Handles authentication
  - Manages API endpoints

HOW IT ALL CONNECTS:

User visits localhost:5000
    ↓
Landing Page (views/index.html)
    ↓ Sign in with Google
Auth via Supabase (public/supabase-auth.js)
    ↓
User Profile stored in database
    ↓
User chooses:

    Option 1: FSAT Test
    → /test (views/test.html)
    → Onboarding captures phone
    → Takes test (views/testscript.js)
    → Results stored in Supabase

    Option 2: Quick Resume
    → /resume (views/resume.html)
    → Simple form
    → Claude AI generates (server.js)
    → Download instantly

    Option 3: Advanced Builder
    → /builder/resumes (frontend/dist/)
    → React UI loads
    → Multiple templates
    → Python service for AI (resume/server.py)
    → Professional output

All authenticated via Supabase
All data stored in same database
